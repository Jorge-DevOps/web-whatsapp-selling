🔧 JENKINS CI/CD PIPELINE
════════════════════════════════════════════════════════════════════════════

Jenkinsfile - 9 Stages automatizados:

  1. 🔍 Checkout (2 min)
     Clona repositorio de GitHub con SSH

  2. 🧹 Clean (1 min)
     Limpia workspace anterior

  3. 📥 Install Dependencies (3 min)
     npm ci --only=production

  4. 🔍 Lint & Validate (1 min)
     Valida sintaxis JavaScript y JSON

  5. 🧪 Test (2 min)
     Tests de validación de código

  6. 🐳 Build Docker Image (5 min)
     Construye imagen con tags
     - jorgde/whatsapp-selling-bot:latest
     - jorgde/whatsapp-selling-bot:BUILD_NUMBER

  7. 🔐 Push to Registry (3 min)
     Sube a Docker Hub (solo rama main)

  8. 📊 Generate Reports (1 min)
     Crea reportes de build

  9. 📦 Archive Artifacts (1 min)
     Archiva para descarga

Tiempo total: ~20 minutos por build

Triggers:
  • GitHub hook (webhook)
  • Poll SCM (cada 5 min)
  • Manual "Build Now"

🚀 FLUJO COMPLETO DE DEPLOYMENT
════════════════════════════════════════════════════════════════════════════

1. Desarrollador hace cambios locales:
   git add .
   git commit -m "feature: nueva funcionalidad"
   git push origin main

2. GitHub webhook dispara Jenkins:
   Automatically triggered

3. Jenkins ejecuta pipeline:
   Checkout → Validate → Build → Test → Docker Build

4. Docker image se sube a Docker Hub:
   docker push jorgde/whatsapp-selling-bot:latest
   docker push jorgde/whatsapp-selling-bot:BUILD_NUMBER

5. Desplegar en servidor:
   docker-compose pull
   docker-compose up -d

6. Bot está corriendo:
   docker logs whatsapp-bot
